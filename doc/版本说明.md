## 版本历史

### v1.0.4 (2026-03-20)

**发布日期**: 2026-03-20

**版本状态**: 补丁修复

**修复内容**:
- 🔧 修复任务创建时报 "Request failed" 错误（Pydantic 序列化问题，ORM 模型无法直接序列化为 JSON）
- 🔧 修复展开任务时附件列表不显示的问题（SQLAlchemy 懒加载导致 attachments 数组为空）
- 🔧 修复附件下载按钮点击无反应的问题（Attachment 模型未导入导致 NameError）
- 🔧 修复图片预览和下载失败的问题（`<img>` 标签无法发送 Authorization header，改用 Blob URL 方式）
- 🔧 修复 Vite 代理无法转发 Authorization header 的问题

**新增内容**:
- ✨ 支持拖拽上传：可将图片/视频文件直接拖入上传区域自动上传
- ✨ 任务展开/折叠：点击附件图标可展开任务，查看附件缩略图和下载按钮
- ✨ 附件下载按钮：在展开视图中为每个附件提供下载按钮

**使用方式**:
```javascript
// 拖拽上传：直接将文件拖入上传区域
// 展开任务：点击任务行的附件数量图标
// 下载附件：展开任务后点击下载按钮
// 图片预览：点击展开视图中的图片缩略图
```

## 版本简介

Memo v1.0.x 是系统的首个正式版本系列，包含了待办任务管理系统的全部核心功能，适用于个人任务管理和小型团队协作场景。

## 部署指南

### 目录结构

```
memo/
├── backend/                 # Python FastAPI 后端
│   ├── app/
│   │   ├── main.py         # 应用入口
│   │   ├── database.py     # 数据库配置
│   │   ├── models/         # 数据模型
│   │   ├── schemas/        # Pydantic schemas
│   │   ├── routers/        # API 路由
│   │   ├── services/       # 业务逻辑
│   │   └── utils/          # 工具函数
│   ├── uploads/            # 附件存储目录
│   ├── logs/               # 日志目录
│   ├── venv/               # Python 虚拟环境
│   ├── requirements.txt
│   └── memo.db             # SQLite 数据库
├── frontend/               # Vue 3 前端
│   ├── src/
│   │   ├── api/            # API 调用
│   │   ├── components/      # 公共组件
│   │   ├── views/          # 页面组件
│   │   ├── router/         # 路由配置
│   │   └── stores/         # 状态管理
│   ├── node_modules/       # 前端依赖
│   ├── package.json
│   └── vite.config.js
├── docker/                 # Docker 配置
├── doc/                    # 文档目录
├── install.sh             # 一键安装脚本
├── start.sh              # 启动服务脚本
├── stop.sh               # 停止服务脚本
├── docker-compose.yml
└── README.md
```

### 快速部署

```bash
# 1. 一键安装（首次部署）
chmod +x install.sh
./install.sh

# 2. 启动服务
./start.sh

# 3. 停止服务
./stop.sh
```

### Docker 部署

```bash
docker-compose up -d
```

## 升级指南

### 从 v1.0.0 升级到 v1.0.1

1. 停止服务：`./stop.sh`
2. 备份数据库和上传文件
3. 替换代码（保留 `backend/venv`、`frontend/node_modules`、`backend/memo.db`）
4. 重新安装依赖（如有必要）：
   ```bash
   rm -rf backend/venv
   ./install.sh
   ```
5. 启动服务：`./start.sh`

### 未来版本升级

升级前请：
1. 备份数据库 (`memo.db`)
2. 备份上传文件 (`uploads/`)
3. 阅读版本升级说明
4. 执行数据库迁移（如有）
